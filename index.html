Gen AI intern take-home (Legixo Thinklabs)

Build a small Q&A HTTP API over documents: Python, LangGraph, Pinecone. Plain language brief below.

Sample documents (download)

We give you a zip of fake legal-style notes (made-up parties and courts). You can use this as your whole corpus, or mix in more files in the same style.

Quick facts

TimeAbout 3–5 days. If you stop early, say what you skipped in the README.
GoalAnswer questions only from your document set. Show which chunk each answer came from. If the docs do not say it, say you cannot find it.
KeysUse your own API keys (LLM + embeddings + Pinecone). Never put keys in git.

What you must use

  • Python (3.10 or newer).
  • LangGraph to run the flow (for example StateGraph). Do not hide everything inside one giant LLM call with no graph.
  • Pinecone for the vector index. Use the real Pinecone service and their Python client. A fake “in-memory only” vector DB does not count.
  • HTTP API only for Q&A: expose at least one endpoint (for example POST /ask) that accepts a question and returns JSON (answer + citations, and optional trace). Use a framework you like (for example FastAPI or Flask). No CLI for asking questions. Ingestion may be a separate documented command or an API route—your choice, but reviewers must be able to load the corpus without a custom REPL.

What you must build

  1. Ingest: Read the files, split into chunks, embed, save vectors in Pinecone with metadata (at least chunk id and source file path) so you can cite them later.
  2. Ask: User asks a question → your LangGraph should search Pinecone, then check if the chunks are good enough (this step should use a branch in the graph: good path vs bad path), then write the answer with citations. Set a max step / max loop so it cannot spin forever.
  3. Docs: README.md (how to install, set env vars, ingest, start the API server, example curl or Postman steps). .env.example with dummy values only. A short LangGraph map in docs/langgraph.md or in the README (list nodes, what each does, simple diagram is fine).
  4. Self-test file: about 10–15 questions in JSON or CSV with columns like question, which files should appear in citations, and your own pass/fail notes after you try them.

Pinecone checklist

  • Tell us how to create the index and which env vars you need (in .env.example).
  • Say what happens if we run ingest twice (for example: same point ids, or delete namespace first).

What to send us

  • Link to your git repo.
  • README with setup and a link to a 5–10 minute video: install, ingest, start the API, call the ask endpoint (for example with curl or Postman), a few good answers with citations, one question the docs cannot answer, and you pointing at your LangGraph layout.

How we will score (simple)

  • Does the LangGraph make sense (clear steps, branch, limit)?
  • Are answers tied to real chunks? Any fake citations?
  • Does Pinecone ingest and search work?
  • Can a new person run the server and call the API from the README?

Extras (not required): LangSmith, hybrid search, reranker.
